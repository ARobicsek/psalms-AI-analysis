# Copy Editor Pass: Psalm Commentary Quality Control

## Purpose

Each psalm commentary is generated by a writer model that receives an extensive research package (medieval commentators, lexical data, LXX readings, liturgical sources, cross-cultural references, Talmudic citations, etc.). The writer produces strong work — detailed Hebrew analysis, substantive engagement with Rashi/Radak/Ibn Ezra/Malbim/Meiri, genuine theological insight. But it produces a small, recurring set of errors that a human editor would catch on a close read. These errors fall into well-defined categories and can be corrected by a second model pass without rewriting the commentary's voice or substance.

This document specifies the error taxonomy, the system prompt for the editor model, the model configuration, and worked examples drawn from actual psalm outputs.

---

## Error Taxonomy

Six categories, ordered by detectability and importance.

### 1. False structural claims

The writer frequently labels rhetorical patterns with technical terms — chiasmus, inclusio, merism, anaphora — that have precise formal definitions. In some cases, the pattern claimed does not match the definition given.

**What to check:**
- **Chiasmus** requires A-B-B'-A' element order. If the elements appear in A-B-A'-B' order, that is parallelism (possibly antithetical parallelism), not chiasmus.
- **Inclusio** requires the same word, phrase, or formula at the opening and closing of a unit. A word that "does double duty" across two cola (governing both without being repeated) is not the same as the word appearing twice. Thematic similarity is not inclusio; verbal repetition is.
- **Merism** requires two poles that together represent a complete spectrum (e.g., "heaven and earth" = everything). Two items from the same domain are not a merism.

**Example (Psalm 37, vv. 14–15):**

The writer claims:

> Verse 15 reverses the scene with chiastic precision (chiasmus: an A-B-B'-A' inversion).

The actual element order:

```
v.14:  sword (A) → bow (B) → targets
v.15:  sword (A') → heart / bows (B') → broken
```

This is A-B-A'-B' — parallelism with agency reversal. The semantic reversal (weapons aimed outward → weapons turned inward) is real and worth noting, but it is not chiasmus. The parenthetical definition ("an A-B-B'-A' inversion") makes the error self-refuting: the text defines chiasmus correctly and then fails its own definition.

**Correction:** Replace "chiastic precision" with "antithetical parallelism" or "point-by-point reversal." Remove the parenthetical defining chiasmus, since chiasmus is no longer being claimed.

### 2. Internal inconsistencies

The writer sometimes makes a claim that contradicts its own earlier analysis of the same verse or word.

**What to check:**
- When a feature is described in the verse-by-verse commentary, does the description match how that feature is characterized in the introduction or in commentary on a later verse?
- When a count is given ("double X," "three occurrences," "five times"), does the actual text support the count?

**Example (Psalm 38, vv. 2 and 22):**

In the commentary on verse 2, the writer correctly observes:

> Ibn Ezra captures this: the verse is structured so that אַל governs both cola, serving double duty.

This accurately describes verse 2, which contains one אַל syntactically governing two clauses.

But in the commentary on verse 22, the writer claims:

> The double אַל ("do not") echoes the opening verse 2's double אַל, establishing an inclusio for the entire psalm.

Verse 2 does not have a "double אַל" — it has one אַל doing double duty, as the writer itself noted. Verse 22 does have two (אַל תַּעַזְבֵנִי... אַל תִּרְחַק). The inclusio observation (both verses open with אַל + plea to YHWH) is valid, but the "double אַל // double אַל" framing is factually wrong and contradicts the earlier analysis.

**Correction:** Rewrite the v. 22 claim to say something like: "The opening אַל echoes verse 2's אַל, establishing an inclusio for the entire psalm — though the closing plea now doubles the negation (אַל... אַל), escalating the urgency."

### 3. Form/content confusion

The writer sometimes claims that the sound, syntax, or morphology of a word or phrase "performs," "enacts," "mimics," or "embodies" a meaning — when the effect actually comes from what the words mean (semantics), not from how they sound or how they are structured (form).

**What to check:**
- Is a specific formal feature identified (a sound pattern, a grammatical construction, a morphological shape)?
- Does the claimed effect genuinely arise from that formal feature, or does it arise from the semantic content of the words?
- Legitimate form-meaning connections: onomatopoeia, iconic reduplication (e.g., סְחַרְחַר where the doubled syllables genuinely mimic the circular motion described), cognate accusative intensification, acrostic constraints that mirror thematic content.
- Illegitimate claims: saying a root "thuds" or "crashes" when the phonemes have no inherent weight; saying syntax "performs" something that the words would convey in any syntactic arrangement.

**Example (Psalm 38, vv. 5–6):**

The writer claims:

> The cognate intensification כָבֵד/יִכְבְּדוּ — "a *heavy* burden, they are *heavy*" — enacts heaviness through sound: the root כבד appears twice, thudding like a stone.

The root repetition (cognate accusative) creates rhetorical emphasis — that is a real formal effect. But the claim that the consonants כ-ב-ד inherently "thud like a stone" is phonosemantic overreach. The emphasis comes from the grammatical construction (cognate accusative), not from the phonetic quality of the consonants.

**Correction:** Remove "thudding like a stone." The sentence can read: "the root כבד appears twice, the cognate intensification hammering the point through grammatical repetition."

**Counterexample — legitimate form-meaning claim (Psalm 38, v. 11):**

> סְחַרְחַר... a reduplicated form... its stuttering doubled syllables mimicking the dizzy spin they describe.

This one works. The pe'al'al reduplication (סְחַרְ-חַר from סחר) is a specific morphological pattern, and the repetition of the syllable genuinely creates a sonic circularity that mirrors the meaning "to go round and round." The formal feature is identified, and the effect belongs to the form.

### 4. Negative or empty citations

The writer occasionally cites a scholar only to report that the scholar does not address the topic at hand, or mentions a commentator as a foil without engaging their actual view.

**What to check:**
- Is the scholar's name followed by a substantive claim, interpretation, or textual observation?
- Or is the scholar mentioned only to say "X does not comment on this," "X is silent here," or "unlike X, who..."?

**Example pattern (not from these specific psalms, but recurring):**

> Ibn Ezra does not comment on this shift, but it likely reflects...

This cites Ibn Ezra to report his silence, then proceeds with the writer's own speculation. The sentence would be stronger without the empty citation: just state the observation directly.

**Correction:** Remove the negative citation. Begin with the writer's own analysis.

**Exception:** A scholar's silence is worth noting when it is itself significant — e.g., when all other major commentators address a crux and one conspicuously does not, and this silence reveals something about their interpretive method. This is rare.

---

### 5. Hebrew script integrity

All Hebrew words in running text must appear in Hebrew characters. Transliterations in parentheses (as pronunciation aids for non-Hebrew readers) are welcome and should be preserved. But if a Hebrew letter name has been substituted with its English equivalent within what should be Hebrew text, restore the Hebrew character.

**What to check:**
- Hebrew words in running text use Hebrew Unicode characters, not romanized equivalents.
- Transliterations appear in parentheses or backticks as aids, not as replacements.
- Vowel pointing (nikkud) is consistent within a given Hebrew word (either fully pointed or unpointed — not a mix).

This error is rare in the current output but has occurred in other psalm runs.

### 6. Weak cross-cultural parallels

The writer draws cross-cultural parallels from classical literature (Homer, Sophocles, Dante, Shakespeare, Ovid), East Asian poetry (Bashō, Laozi), and modern Hebrew poetry. These range from genuinely illuminating to surface-level.

**What to check:**
- Does the parallel share substantive theological, structural, or thematic connection with the psalm — or only a visual/surface similarity?
- Does the commentary itself have to explain why the parallel doesn't actually hold? If the writer introduces a parallel and then immediately walks it back ("But where X does Y, the psalmist does the opposite..."), the parallel is not earning its place.
- A parallel that illuminates the psalm's distinctive move by contrast ("unlike X, the psalmist does Y because...") is valid only if the contrast reveals something the reader wouldn't have seen without the comparison.

**Strong parallel — keep (Psalm 38, vv. 5–6):**

> Sophocles' Philoctetes, abandoned on Lemnos with a festering snakebite, asks the same terrible question: πῶς γὰρ ἐξηνέσχετο τὴν ἐμὴν δυσώδειαν, "How could he endure my foul stench?" (*Philoctetes*, 874–875). But Philoctetes' wound is arbitrary fate; the psalmist attributes his festering to אִוַּלְתִּי, "my folly."

This works because: (a) the physical parallel is exact (festering wound, stench, social isolation), (b) the theological difference (arbitrary fate vs. moral causation) sharpens the psalm's distinctive claim, and (c) the contrast genuinely illuminates rather than merely decorating.

**Weak parallel — remove (Psalm 38, v. 7):**

> Dante's *Purgatorio* offers a visual counterpart: the proud souls crushed under enormous stones, giugner le ginocchia al petto, "joining knees to chest" (X.131) — bodies bent double under the weight of their own spiritual failings. But Dante's punishment has a terminus; the psalmist walks his circuit of gloom without a visible exit.

The connection is "bent bodies" — a visual similarity. Dante's theological framework (purgatorial discipline with a terminus) is fundamentally different, as the writer acknowledges. The etymological wordplay (עוה/עוון) that makes v. 7 distinctive has no counterpart in Dante. The parallel adds decoration but not understanding.

## What the Editor Should NOT Do

- **Do not rewrite for style or voice.** The commentary has a distinctive authorial register — sometimes lyrical, sometimes clinical. Preserve it.
- **Do not remove parenthetical term definitions** (e.g., "(the ancient Greek translation of the Hebrew Bible)" after LXX, "(a framing device where a unit begins and ends with the same word or phrase)" after inclusio). Readers may encounter only one psalm guide, so these definitions serve a real audience need. Remove a definition only if the identical term has already been defined earlier in the same psalm's commentary.
- **Do not add material.** The editor's job is subtractive and corrective, not generative.
- **Do not remove content that is merely debatable.** Scholarly interpretation involves judgment calls. If a reading is defensible even if not universally accepted, leave it. Only correct claims that are demonstrably wrong by the text's own standards.
- **Do not challenge the accuracy of research-sourced citations.** Talmud folio references, medieval commentator quotations, LXX readings, liturgical practice descriptions, and cross-cultural quotes come from a curated research package and should be treated as accurate. (If something looks obviously garbled — e.g., a Hebrew word missing characters — fix the surface error, but do not second-guess the citation's substance.)
- **Do not remove or restructure the liturgical sections.** These sections (marked by `---LITURGICAL-SECTION-START---`) have their own internal logic and should be edited with the same rules but not reorganized.

## System Prompt

```
You are a copy editor for scholarly psalm commentary written for an educated
general audience. Make minimal, targeted corrections. Do not rewrite for
style or voice. Do not add material. Do not remove content that is merely
debatable — only correct content that is demonstrably wrong, empty, or
self-contradicting.

Preserve parenthetical definitions of technical terms (e.g., defining
"inclusio," "LXX," "Hiphil," "chiasmus" on first use within a psalm
commentary). Readers may be reading only one psalm guide. Remove a
definition only if the same term was already defined earlier in the same
psalm's commentary.

Correct the following categories of error:

1. STRUCTURAL CLAIMS. If the text labels a pattern as chiasmus, verify
   A-B-B'-A' element order in the cited text. If the order is A-B-A'-B',
   relabel it as parallelism or antithetical parallelism as appropriate.
   If inclusio is claimed, verify the opening and closing elements are
   the same word or phrase actually appearing in both positions — a word
   doing "double duty" across two cola is not the same as the word
   appearing twice. If merism is claimed, verify the two named poles
   represent a genuine complete spectrum. Correct the label or remove the
   claim. Remove any parenthetical definition that now contradicts the
   corrected label.

2. INTERNAL INCONSISTENCIES. If a claim contradicts something stated
   earlier in the same commentary (e.g., describing a verse as having a
   "double X" when the earlier analysis showed it has one X doing double
   duty), correct the later statement to be consistent with the earlier,
   more detailed analysis.

3. FORM/CONTENT CONFUSION. If the text claims that sound, syntax,
   morphology, or grammar "performs," "enacts," "mimics," or "embodies"
   a meaning, check two things: (a) is a specific formal feature
   identified? (b) does the effect genuinely arise from that formal
   feature rather than from what the words mean? If the meaning comes
   from semantics rather than form, rewrite the sentence to attribute the
   effect correctly. Do not remove the observation — just reattribute it.
   Note: reduplication (pe'al'al forms), cognate accusatives, and
   onomatopoeia are legitimate form-meaning connections when correctly
   identified.

4. NEGATIVE CITATIONS. If a scholar is cited only to report that they
   do not address the topic, or is mentioned only as a foil without
   engaging their actual view, remove the sentence. Exception: retain
   if the silence is itself the point of analysis (e.g., a conspicuous
   absence among commentators who all address a crux).

5. HEBREW SCRIPT. Hebrew words in running text must use Hebrew Unicode
   characters. Transliterations in parentheses as pronunciation aids
   are fine and should be preserved. If a Hebrew letter name has been
   substituted for the character itself within Hebrew text, restore the
   character. Fix inconsistent vowel pointing within a single word.

6. WEAK CROSS-CULTURAL PARALLELS. If a literary parallel from outside
   the biblical/rabbinic tradition is introduced the connection should feel strong and not forced. Keep parallels where the connection is substantive and
   the contrast (if any) genuinely illuminates the psalm's distinctive
   theological move.

After the corrected text, append a "Changes" section listing each change
with its category number and a one-line explanation.
```

## Model Configuration

### Model
**Claude Opus 4.6** (`claude-opus-4-6`)

Rationale: The task requires sensitivity to Hebrew grammatical structures (verifying whether a chiasmus claim holds requires parsing element order in biblical Hebrew), rabbinic citation patterns, and the difference between formal and semantic effects. Opus handles these reliably. The cost difference over smaller models (~$0.08/psalm) is negligible at this volume.

### Thinking
**Enabled, budget: 4,096 tokens**

Rationale: Rules 1 and 2 (structural claims, internal consistency) require multi-step reasoning:
- Parse the structural claim
- Identify the elements in the biblical text
- Lay them out in sequence
- Check whether the claimed pattern holds

The chiasmus error in Psalm 37:14–15 requires four reasoning steps to catch. The internal inconsistency in Psalm 38:2/22 requires comparing claims separated by ~3,000 tokens. Light thinking makes these checks reliable without significant cost increase.

Thinking adds ~$0.02–0.04 per psalm at 4K budget. Total per-psalm cost with thinking: **~$0.17–0.20**.

### Temperature
**0.0** (or as low as the API allows)

The editor should be deterministic. Creative variation is undesirable in a correction pass.

### Max output tokens
**16,384** (or match input length + 1,024 for the Changes section)

The editor returns the full corrected text plus a short appendix. Output length ≈ input length.

## API Call Structure

```python
import anthropic

client = anthropic.Anthropic()

def run_copy_editor(psalm_text: str) -> str:
    response = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=16384,
        temperature=0,
        thinking={
            "type": "enabled",
            "budget_tokens": 4096
        },
        system=SYSTEM_PROMPT,  # The system prompt from the section above
        messages=[
            {
                "role": "user",
                "content": psalm_text
            }
        ]
    )

    # Extract text blocks (skip thinking blocks)
    result = ""
    for block in response.content:
        if block.type == "text":
            result += block.text

    return result
```

### Input format
Pass the full psalm commentary (introduction + liturgical section + verse-by-verse) as a single user message. No additional framing or instruction is needed in the user message — the system prompt contains all rules.

### Output format
The model returns:
1. The full corrected text (preserving all formatting, headers, Hebrew text, section markers)
2. A `## Changes` section at the end, listing each correction with its category number and a one-line explanation

## Workflow

1. **Generate** the psalm commentary using the writer model with its full research package.
2. **Run the copy editor** by passing the complete output through the editor model using the configuration above.
3. **Review the Changes section** at the bottom of the editor's output. Each change is categorized and explained in one line.
4. **Accept or reject** individual changes. The Changes section is a manifest, not a mandate — the human author has final say.
5. **Strip the Changes section** before publication.

### Spot-check protocol

For the first 10 psalms, manually verify every change the editor makes. This calibrates trust in the editor's judgment and catches any systematic over- or under-correction. After calibration, spot-check every 5th psalm.

---

## Limitations

This editor will **not** catch:

- **Theological interpretive errors** — if the writer misreads a commentator's position but does so fluently, the editor has no independent access to the source text to check.
- **Missing content** — the editor is subtractive/corrective. If the writer omitted an important observation, the editor will not add it.
- **Systematic biases** — if the writer consistently over-represents one commentator or under-represents another, the editor will not notice, since it sees only one psalm at a time.
- **Errors in the research package** — citations sourced from the research package are treated as accurate by design.

These limitations are acceptable for a light editorial pass. They define the boundary between automated quality control and human scholarly review.
